Don't know how to connect to HTB VPN & RDP? Click here.

Download the VPN configuration file from HTB Academy and run the following command in the same directory:

sudo openvpn academy-regular.ovpn # Make sure OpenVPN is installed.

Command Breakdown

  • sudo → Runs the command with administrator (root) privileges.
  • openvpn → Starts the OpenVPN client.
  • academy-regular.ovpn → The VPN configuration file that contains the settings required to connect to the HTB Academy VPN.

Once the VPN connection is established, connect to the spawned target machine using Remote Desktop Protocol (RDP):

xfreerdp /v:<ip-address> /u:htb-student /p:'Academy_WinFun!' # Install xfreerdp if needed.

Command Breakdown

  • xfreerdp → A free and open-source RDP client for Linux.
  • /v:<ip-address> → Specifies the IP address of the target Windows machine.
  • /u:htb-student → The username used to log in.
  • /p:'HTB_@cademy_stdnt!' → The password for the user account.

After successfully authenticating, you will be presented with the desktop of the target Windows machine.

What system user has full control over the C:\Users directory?

To determine which user has full control over the C:\Users directory, use the following command:

icacls C:\Users

Command Breakdown

  • icacls → A built-in Windows command-line utility used to view and modify Access Control Lists (ACLs) on files and directories. It can display existing permissions, grant or deny permissions, back up ACLs, restore ACLs, and remove permissions. It is commonly used by both system administrators and penetration testers to audit file and folder permissions.
  • C:\Users → Specifies the directory whose permissions you want to inspect. This directory contains the user profile folders for every account on the Windows machine.

What are Access Control Lists (ACLs)?

An Access Control List (ACL) is a list of permissions attached to a file or directory that determines which users and groups can access the resource and what actions they are allowed to perform.

Each entry within an ACL is called an Access Control Entry (ACE). An ACE specifies:

  • The user or group.
  • The permissions assigned to that user or group.

Windows uses ACLs to enforce security on NTFS file systems.

Understanding the Output

Running the command produces output similar to:

BUILTIN\Administrators:(F)
NT AUTHORITY\SYSTEM:(F)
Users:(RX)

Each permission is represented by a letter or combination of letters enclosed in parentheses. Some common permission flags include:

  • (F) – Full Control
  • (M) – Modify
  • (RX) – Read and Execute
  • (R) – Read
  • (W) – Write

For this question, look specifically for the user or group that has the (F) permission. The user or group associated with the (F) permission is the answer you need to submit.

Note: icacls is one of the primary Windows utilities for managing NTFS permissions. During penetration testing, it is frequently used to identify misconfigured permissions that could potentially be abused for privilege escalation or unauthorized access.

Don't know how to connect to HTB VPN & RDP? Click here.

Download the VPN configuration file from HTB Academy and run the following command in the same directory:

sudo openvpn academy-regular.ovpn # Make sure OpenVPN is installed.

Command Breakdown

  • sudo → Runs the command with administrator (root) privileges.
  • openvpn → Starts the OpenVPN client.
  • academy-regular.ovpn → The VPN configuration file that contains the settings required to connect to the HTB Academy VPN.

Once the VPN connection is established, connect to the spawned target machine using Remote Desktop Protocol (RDP):

xfreerdp /v:<ip-address> /u:htb-student /p:'Academy_WinFun!' # Install xfreerdp if needed.

Command Breakdown

  • xfreerdp → A free and open-source RDP client for Linux.
  • /v:<ip-address> → Specifies the IP address of the target Windows machine.
  • /u:htb-student → The username used to log in.
  • /p:'HTB_@cademy_stdnt!' → The password for the user account.

After successfully authenticating, you will be presented with the desktop of the target Windows machine.

Find the SID of the bob.smith user.

To retrieve the Security Identifier (SID) of the bob.smith user, run:

wmic useraccount where "name='bob.smith'" get sid

Command Breakdown

  • wmic → Launches the Windows Management Instrumentation Command-line (WMIC) utility.
  • useraccount → Specifies the WMI class that contains information about local user accounts.
  • where → Filters the query results based on the specified condition.
  • "name='bob.smith'" → Restricts the query to the user account named bob.smith.
  • get → Displays the specified property from the query result.
  • sid → Retrieves the Security Identifier (SID) of the matching user account.

A Security Identifier (SID) is a unique value assigned to every user, group, or computer account in Windows. Windows uses SIDs internally to identify security principals instead of account names.

Submit the displayed SID as your answer.

What 3rd party security application is disabled at startup for the current user? (The answer is case sensitive.)

To view the applications configured to start automatically for the current user, inspect the Run registry key:

Get-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Run"

Command Breakdown

  • Get-ItemProperty → Retrieves the properties and values stored in a registry key.
  • -Path → Specifies the registry key to query.
  • HKCU: → Abbreviation for HKEY_CURRENT_USER, a registry hive containing settings specific to the currently logged-in user.
  • Software\Microsoft\Windows\CurrentVersion\Run → The registry key that stores applications configured to execute automatically when the current user logs on.

Review the listed startup entries and identify the third-party security application. Submit its name exactly as shown.

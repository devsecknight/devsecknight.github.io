Don't know how to connect to HTB VPN & RDP? Click here.

Download the VPN configuration file from HTB Academy and run the following command in the same directory:

sudo openvpn academy-regular.ovpn # Make sure OpenVPN is installed.

Command Breakdown

  • sudo → Runs the command with administrator (root) privileges.
  • openvpn → Starts the OpenVPN client.
  • academy-regular.ovpn → The VPN configuration file that contains the settings required to connect to the HTB Academy VPN.

Once the VPN connection is established, connect to the spawned target machine using Remote Desktop Protocol (RDP):

xfreerdp /v:<ip-address> /u:htb-student /p:'Academy_WinFun!' # Install xfreerdp if needed.

Command Breakdown

  • xfreerdp → A free and open-source RDP client for Linux.
  • /v:<ip-address> → Specifies the IP address of the target Windows machine.
  • /u:htb-student → The username used to log in.
  • /p:'HTB_@cademy_stdnt!' → The password for the user account.

After successfully authenticating, you will be presented with the desktop of the target Windows machine.

What is the Build Number of the target workstation?

The Get-WmiObject cmdlet retrieves information from Windows Management Instrumentation (WMI).

Windows Management Instrumentation (WMI) is a Windows management framework that provides access to information about the operating system, hardware, installed software, services, and other system components.

In this command, the Win32_OperatingSystem WMI class is queried to retrieve information about the installed operating system. The output is then filtered to display only the BuildNumber property, which represents the build number of the installed Windows operating system.

Get-WmiObject -Class Win32_OperatingSystem | Select BuildNumber

Command Breakdown

  • Get-WmiObject → Retrieves management information from Windows Management Instrumentation (WMI).
  • -Class → Specifies the WMI class to query.
  • Win32_OperatingSystem → A WMI class that contains information about the installed Windows operating system.
  • | → The pipeline operator. Passes the output of the first command as input to the next command.
  • Select → Displays only the specified property from the command output.
  • BuildNumber → The operating system's build number.

Submit the value of the BuildNumber property as your answer.

Which Windows NT version is installed on the workstation? (i.e. Windows X - case sensitive)

The same WMI class can also be used to retrieve the Windows version. Instead of displaying the BuildNumber property, this command selects the Version property, which contains the Windows NT version number of the installed operating system.

Get-WmiObject -Class Win32_OperatingSystem | Select Version

Command Breakdown

  • Get-WmiObject → Retrieves management information from Windows Management Instrumentation (WMI).
  • -Class → Specifies the WMI class to query.
  • Win32_OperatingSystem → A WMI class that contains information about the installed Windows operating system.
  • | → The pipeline operator. Passes the output of the first command as input to the next command.
  • Select → Displays only the specified property from the command output.
  • Version → Displays the Windows NT version of the installed operating system.

Note: The Version property returns the Windows NT version number rather than the Windows marketing name. If the challenge expects the Windows release name (for example, Windows 10), map the NT version to its corresponding Windows release before submitting your answer.

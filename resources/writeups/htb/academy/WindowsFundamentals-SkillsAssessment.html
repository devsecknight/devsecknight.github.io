Don't know how to connect to HTB VPN & RDP? Click here.

Download the VPN configuration file from HTB Academy and run the following command in the same directory:

sudo openvpn academy-regular.ovpn # Make sure OpenVPN is installed.

Command Breakdown

  • sudo → Runs the command with administrator (root) privileges.
  • openvpn → Starts the OpenVPN client.
  • academy-regular.ovpn → The VPN configuration file that contains the settings required to connect to the HTB Academy VPN.

Once the VPN connection is established, connect to the spawned target machine using Remote Desktop Protocol (RDP):

xfreerdp /v:<ip-address> /u:htb-student /p:'Academy_WinFun!' # Install xfreerdp if needed.

Command Breakdown

  • xfreerdp → A free and open-source RDP client for Linux.
  • /v:<ip-address> → Specifies the IP address of the target Windows machine.
  • /u:htb-student → The username used to log in.
  • /p:'HTB_@cademy_stdnt!' → The password for the user account.

After successfully authenticating, you will be presented with the desktop of the target Windows machine.

Required Steps

Creating a Shared Folder Named Company Data

  1. On the Desktop, create a new folder named Company Data.
  2. Right-click the folder and select Properties.
  3. Open the Sharing tab.
  4. Click Advanced Sharing.
  5. Check Share this folder, then click Apply and OK.

Creating a Subfolder Named HR

Open the Company Data folder and create a new subfolder named HR.

Creating a User Named Jim

  1. Right-click the Start button and select Computer Management.
  2. Navigate to Local Users and Groups → Users.
  3. Right-click inside the right pane and select New User.
  4. Create a user named Jim.
  5. Uncheck User must change password at next logon.
  6. Click Create.

Creating a Security Group Named HR

  1. In Computer Management, navigate to Local Users and Groups → Groups.
  2. Right-click and select New Group.
  3. Enter HR as the group name.
  4. Click Create.

A security group is a collection of user accounts that can be assigned permissions as a single entity, simplifying access management.

Adding Jim to the HR Security Group

  1. Open the HR group's properties.
  2. Click Add.
  3. Enter Jim under Enter the object names to select.
  4. Click Check Names to resolve the account.
  5. Click OK.

Configuring Share Permissions for Company Data

  1. Right-click Company Data and select Properties.
  2. Open the Sharing tab and click Advanced Sharing.
  3. Click Permissions.
  4. Remove the default Everyone group.
  5. Click Add, enter HR, and click Check Names.
  6. Grant the following Allow permissions: Change and Read.
  7. Click Apply and OK.

Share permissions control access to folders when they are accessed over the network using SMB.

Configuring NTFS Permissions for Company Data

  1. Open the Security tab.
  2. Click Advanced.
  3. Click Disable inheritance.
  4. Select Remove all inherited permissions from this object.
  5. Click Add, select the HR group, and grant the following basic permissions: Modify, Read & execute, List folder contents, Read, Write.
  6. Click OK, then Apply.

NTFS permissions control access to files and folders stored on an NTFS-formatted drive. Unlike Share permissions, they apply whether the resource is accessed locally or over the network.

Configuring NTFS Permissions for the HR Subfolder

Repeat the same process for the HR subfolder:

  1. Open Properties → Security → Advanced.
  2. Click Disable inheritance.
  3. Choose Remove all inherited permissions from this object.
  4. Add the HR security group.
  5. Grant the following permissions: Modify, Read & execute, List folder contents, Read, Write.
  6. Click Apply and OK.

What is the name of the group that is present in the Company Data Share Permissions ACL by default?

By default, the Everyone group is present in the Share Permissions Access Control List (ACL).

What is the name of the tab that allows you to configure NTFS permissions?

NTFS permissions are configured from the Security tab.

What is the name of the service associated with Windows Update?

To locate the Windows Update service, run:

Get-Service | Where-Object {$_.DisplayName -like "*Windows Update*"}

Command Breakdown

  • Get-Service → Retrieves all services installed on the system.
  • | → Passes the output of Get-Service to the next command.
  • Where-Object → Filters objects based on a specified condition.
  • $_.DisplayName → Refers to the DisplayName property of each service object currently being processed.
  • -like → Performs wildcard pattern matching.
  • "*Windows Update*" → Matches services whose display name contains Windows Update.

The command displays the Windows Update service. Submit its service name as your answer.

List the SID associated with the user account Jim.

Get-LocalUser -Name Jim | Select-Object SID

Command Breakdown

  • Get-LocalUser → Retrieves local user accounts on the system.
  • -Name Jim → Specifies the user account named Jim.
  • | → Passes the user object to the next command.
  • Select-Object → Displays only the specified properties from an object.
  • SID → The Security Identifier (SID) assigned to the user account.

A Security Identifier (SID) is a unique value assigned to every user, group, or computer account in Windows. Windows uses SIDs internally to identify security principals rather than relying on account names.

Submit the displayed SID as your answer.

List the SID associated with the HR security group.

Get-LocalGroup -Name HR | Select-Object SID

Command Breakdown

  • Get-LocalGroup → Retrieves local security groups on the system.
  • -Name HR → Specifies the HR security group.
  • | → Passes the group object to the next command.
  • Select-Object → Displays only the specified properties from an object.
  • SID → The Security Identifier (SID) assigned to the security group.

Submit the displayed SID as your answer.
